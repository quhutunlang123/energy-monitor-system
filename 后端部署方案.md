## 1. 部署架构概述

### 1.1 系统架构

本系统采用Spring Cloud微服务架构，包含以下核心服务：

```
┌─────────────────────────────────────────────────────────────┐
│                        前端 (Vue.js/React)                  │
│                      http://localhost:8080                   │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   API网关 (Spring Cloud Gateway)            │
│                       http://localhost:8088                  │
└─────────────────────────────────────────────────────────────┘
                              │
          ┌───────────────────┼───────────────────┐
          ▼                   ▼                   ▼
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│  认证服务       │  │  用户服务       │  │  设备服务       │
│  (auth-service) │  │  (user-service) │  │  (device-service)│
│  :8081          │  │  :8082          │  │  :8083           │
└─────────────────┘  └─────────────────┘  └─────────────────┘
          │                   │                   │
          └───────────────────┼───────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   Nacos (服务注册与配置中心)                 │
│                       http://localhost:8848                  │
└─────────────────────────────────────────────────────────────┘
                              │
          ┌───────────────────┼───────────────────┐
          ▼                   ▼                   ▼
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│     MySQL       │  │     Redis       │  │    MongoDB      │
│     :3306       │  │     :6379       │  │    :27017       │
└─────────────────┘  └─────────────────┘  └─────────────────┘
```

## 3. Docker Compose部署

### 3.2 创建 docker-compose.yml

> 说明：本项目使用阿里云私有仓库存储Docker镜像，以提高镜像拉取速度和稳定性。

```yaml
version: '3.8'

services:

  mysql:
    image: crpi-880y0pjuczeqxg2h.cn-hangzhou.personal.cr.aliyuncs.com/quhu-images/mysql:8.0
    container_name: energy-mysql
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: energy_monitor
      TZ: Asia/Shanghai
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql
      - ./mysql-init.sql:/docker-entrypoint-initdb.d/01-mysql-init.sql
    networks:
      - energy-network
    restart: unless-stopped

  # Redis缓存
  redis:
    image: crpi-880y0pjuczeqxg2h.cn-hangzhou.personal.cr.aliyuncs.com/quhu-images/redis:7-alpine
    container_name: energy-redis
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    networks:
      - energy-network
    restart: unless-stopped

  # Nacos服务注册与配置中心
  nacos:
    image: crpi-880y0pjuczeqxg2h.cn-hangzhou.personal.cr.aliyuncs.com/quhu-images/nacos-server:v2.2.3
    container_name: energy-nacos
    environment:
      MODE: standalone
      JVM_XMS: 512m
      JVM_XMX: 512m
    ports:
      - "8848:8848"
      - "9848:9848"
    volumes:
      - nacos-logs:/home/nacos/logs
    networks:
      - energy-network
    restart: unless-stopped
    depends_on:
      - mysql
      - redis

networks:
  energy-network:
    driver: bridge

volumes:
  mysql-data:
  redis-data:
  nacos-logs:
```

#### 3.2.1 阿里云私有仓库配置

**为什么使用阿里云私有仓库？**
- 国内网络环境下，从Docker Hub拉取镜像速度较慢
- 阿里云私有仓库提供稳定的镜像存储和拉取服务
- 支持镜像的版本管理和访问控制

**使用前准备：**
1. 登录阿里云容器镜像服务控制台
2. 创建个人或企业版镜像仓库
3. 推送基础镜像到私有仓库
4. 在部署机器上登录阿里云Docker Registry：

```bash
# 登录阿里云Docker Registry
docker login --username=用户名 crpi-880y0pjuczeqxg2h.cn-hangzhou.personal.cr.aliyuncs.com

# 输入密码（阿里云访问凭证）
```

**镜像推送示例：**

```bash
# 拉取官方镜像
docker pull mysql:8.0

# 标记镜像
docker tag mysql:8.0 crpi-880y0pjuczeqxg2h.cn-hangzhou.personal.cr.aliyuncs.com/quhu-images/mysql:8.0

# 推送镜像
docker push crpi-880y0pjuczeqxg2h.cn-hangzhou.personal.cr.aliyuncs.com/quhu-images/mysql:8.0
```

### 3.5 Docker数据卷与持久化

#### 3.5.1 数据卷说明

Docker Compose配置中定义了以下数据卷：

| 数据卷名称 | 挂载路径 | 用途 |
|-----------|----------|------|
| mysql-data | /var/lib/mysql | MySQL数据持久化 |
| redis-data | /data | Redis数据持久化 |
| nacos-logs | /home/nacos/logs | Nacos日志存储 |

#### 3.5.2 数据持久化原理

- **MySQL数据持久化**：通过挂载 `mysql-data` 数据卷到 `/var/lib/mysql` 目录，确保数据库数据在容器重启后不丢失
- **Redis数据持久化**：通过挂载 `redis-data` 数据卷到 `/data` 目录，确保Redis数据在容器重启后不丢失
- **Nacos日志**：通过挂载 `nacos-logs` 数据卷到 `/home/nacos/logs` 目录，便于查看和分析日志

#### 3.5.3 数据卷管理

**查看数据卷：**

```bash
# 查看所有数据卷
docker volume ls

# 查看特定数据卷详情
docker volume inspect mysql-data
```

**备份数据卷：**

```bash
# 备份MySQL数据
docker run --rm -v mysql-data:/volume -v $(pwd):/backup alpine tar -czf /backup/mysql-backup.tar.gz -C /volume .

# 备份Redis数据
docker run --rm -v redis-data:/volume -v $(pwd):/backup alpine tar -czf /backup/redis-backup.tar.gz -C /volume .
```

**恢复数据卷：**

```bash
# 恢复MySQL数据
docker run --rm -v mysql-data:/volume -v $(pwd):/backup alpine tar -xzf /backup/mysql-backup.tar.gz -C /volume

# 恢复Redis数据
docker run --rm -v redis-data:/volume -v $(pwd):/backup alpine tar -xzf /backup/redis-backup.tar.gz -C /volume
```

#### 3.5.4 注意事项

1. **数据卷安全**：定期备份数据卷，防止数据丢失
2. **权限管理**：确保数据卷目录权限正确，避免权限问题
3. **空间管理**：定期清理不需要的日志和数据，避免磁盘空间不足
4. **迁移注意**：在迁移服务时，需要同时迁移数据卷数据

> 说明：上面的 Compose 只负责启动 MySQL / Redis / Nacos 三个基础组件。
> 实际业务库 `energy_monitor` 的表结构可以通过 `mysql-init.sql` 自动初始化（首次启动 MySQL 时执行）。
> Nacos 这里使用 `standalone` 模式（内置存储）以降低部署复杂度；如需将 Nacos 配置落到 MySQL，
> 再按 Nacos 官方文档配置 `nacos_config` 库与表结构。

## 4. Nacos配置

### 4.1 登录Nacos

1. 打开浏览器访问：http://localhost:8848/nacos
2. 用户名：nacos
3. 密码：nacos

### 4.2 创建命名空间

1. 登录后在左侧菜单找到"命名空间"
2. 点击"新建命名空间"
3. 输入：
   - 命名空间ID：dev
   - 命名空间描述：开发环境

### 4.3 配置数据源

#### 4.3.1 配置命名规范

在Nacos中创建配置时，建议使用以下命名规范：

| 配置项 | 命名规则 | 示例 |
|--------|----------|------|
| 命名空间 | 环境标识 | dev（开发环境）、test（测试环境）、prod（生产环境） |
| Data ID | 服务名-环境.yml | auth-service-dev.yml、user-service-dev.yml |
| Group | 项目标识 | ENERGY_PLATFORM |

#### 4.3.2 创建数据库配置

1. 登录Nacos控制台：http://localhost:8848/nacos
2. 选择左侧菜单"配置管理" → "配置列表"
3. 选择命名空间：`dev`
4. 点击"+"按钮新建配置
5. 填写配置信息：
   - Data ID：`datasource-dev.yml`
   - Group：`ENERGY_PLATFORM`
   - 配置格式：`YAML`
   - 配置内容：

```yaml
spring:
  datasource:
    url: jdbc:mysql://mysql:3306/energy_monitor?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: root123
    driver-class-name: com.mysql.cj.jdbc.Driver

  data:
    redis:
      host: redis
      port: 6379
```

#### 4.3.3 为各服务创建配置

为每个微服务创建独立的配置文件：

**认证服务配置**
- Data ID：`auth-service-dev.yml`
- Group：`ENERGY_PLATFORM`

```yaml
server:
  port: 8081

spring:
  application:
    name: auth-service

        namespace: dev
      config:
        server-addr: nacos:8848
        namespace: dev
        group: ENERGY_PLATFORM
        file-extension: yml

# 认证相关配置
auth:
  jwt:
    secret: energy_monitor_secret_key
    expiration: 86400
```

```yaml
  port: 8088
```

#### 4.3.4 配置管理最佳实践

1. **环境隔离**：使用不同的命名空间区分开发、测试、生产环境
2. **服务隔离**：为每个服务创建独立的配置文件
3. **配置分层**：将通用配置（如数据源）与服务特定配置分离
4. **版本管理**：定期备份Nacos配置，使用配置历史记录
5. **安全管理**：敏感配置（如数据库密码）可以考虑使用Nacos的加密功能

#### 4.3.5 与Docker Compose的对应关系

- 数据库连接地址中的`mysql`对应Docker Compose中的服务名
- Redis连接地址中的`redis`对应Docker Compose中的服务名
- Nacos连接地址中的`nacos`对应Docker Compose中的服务名
- 所有服务都在同一个`energy-network`网络中，可以通过服务名直接访问

## 7. 常见问题

### 7.1 Nacos启动失败

- 检查内存是否足够（至少2GB）
- 检查端口8848是否被占用
- 查看日志：`docker compose logs nacos`

### 7.2 服务无法注册到Nacos

- 检查Nacos是否启动成功
- 检查网络配置是否正确
- 检查防火墙端口是否开放

### 7.3 数据库连接失败

- 检查MySQL是否启动成功
- 检查用户名密码是否正确
- 检查数据库是否创建

## 9. Jenkins持续集成部署（可选）

### 9.1 Jenkins简介

Jenkins是一个开源的持续集成/持续部署（CI/CD）工具，可以自动化构建、测试和部署项目。

对于本项目，Jenkins可以实现：
- 代码提交后自动构建
- 自动运行测试
- 自动打包部署到服务器
- 支持多环境部署（开发、测试、生产）

### 9.2 部署架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   GitHub    │────▶│   Jenkins   │────▶│  Docker Hub │
└─────────────┘     └─────────────┘     └─────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │  部署服务器  │
                    │ (Docker环境) │
                    └─────────────┘
```

### 9.3 Jenkins部署

#### 9.3.1 Docker Compose中添加Jenkins

```yaml
  jenkins:
    image: jenkins/jenkins:lts
    container_name: energy-jenkins
    privileged: true
    user: root
    ports:
      - "8080:8080"
      - "50000:50000"
    volumes:
      - jenkins-data:/var/jenkins_home
      - /var/run/docker.sock:/var/run/docker.sock
    networks:
      - energy-network
    restart: unless-stopped
```

#### 9.3.2 启动Jenkins

```bash
docker compose up -d jenkins

# 查看Jenkins日志
docker compose logs -f jenkins
```

#### 9.3.3 初始化Jenkins

1. 打开浏览器访问：http://localhost:8080
2. 获取初始密码：`docker exec energy-jenkins cat /var/jenkins_home/secrets/initialAdminPassword`
3. 安装推荐插件
4. 创建管理员账户

### 9.4 配置Pipeline

#### 9.4.1 创建Pipeline任务

1. 在Jenkins中创建新任务，选择"Pipeline"
2. 配置GitHub仓库地址
3. 编写Jenkinsfile

#### 9.4.2 Jenkinsfile示例

```groovy
pipeline {
    agent any
    
    environment {
        DOCKER_IMAGE = 'energy-platform'
        REGISTRY = 'docker.io'
    }
    
    stages {
        stage('Checkout') {
            steps {
                checkout scm
            }
        }
        
        stage('Build') {
            steps {
                sh 'mvn clean package -DskipTests'
            }
        }
        
        stage('Docker Build') {
            steps {
                sh '''
                    docker build -t ${DOCKER_IMAGE}:${BUILD_NUMBER} ./backend
                    docker tag ${DOCKER_IMAGE}:${BUILD_NUMBER} ${DOCKER_IMAGE}:latest
                '''
            }
        }
        
        stage('Deploy') {
            steps {
                sh '''
                    docker compose -f docker-compose.yml down
                    docker compose -f docker-compose.yml up -d
                '''
            }
        }
    }
    
    post {
        always {
            cleanWs()
        }
    }
}
```

### 9.5 自动触发构建

1. 在GitHub仓库中设置Webhook
2. 每次代码推送自动触发Jenkins构建
3. 配置邮件通知构建结果
